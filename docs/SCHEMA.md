# Pod Schema Reference

Full specification for the T1–T4 pod format used in the TradingGym AI datanet.

---

## T1 — Executed Trades

Every real trade entry and exit. This is your ground truth.

### Open record
```json
{
  "type": "open",
  "ts": "2026-04-18T14:32:11.000000+00:00",
  "agent_id": "your-agent-id",
  "agent_name": "YourBotName",
  "pair": "BTC",
  "side": "long",
  "entry_price": 84500.0,
  "size_usd": 150.0,
  "leverage": 5,
  "score": 6.2,
  "strategy": "swing",
  "signals": {
    "rsi_1h": 28.4,
    "obi": 0.61,
    "atr_pct": 0.82,
    "reasons": ["RSI oversold 1H=28", "OBI strong buy 0.61", "4H trend BULLISH"]
  },
  "take_profit_pct": 0.035,
  "stop_loss_pct": null
}
```

## T2 — Near-Misses (Close Calls)

Signals evaluated but not traded. **Most valuable tier for training.**

```json
{
  "ts": "2026-04-18T14:38:22.000000+00:00",
  "agent_id": "your-agent-id",
  "agent_name": "YourBotName",
  "pair": "ETH",
  "side": "long",
  "score": 4.8,
  "threshold": 5.5,
  "tier": 2,
  "reason_not_traded": "score_4.8_below_threshold_5.5",
  "signals": {
    "rsi_1h": 32.1,
    "obi": 0.44,
    "atr_pct": 1.12,
    "reasons": ["RSI mild oversold", "OBI mild buy", "EMA stack mixed", "VWAP below price"]
  }
}
```

**Why T2 matters:** Near-misses show what the bot *considered* and *why it passed*. Combined with actual market outcomes, this counterfactual signal is extremely valuable for threshold calibration and strategy refinement.

## T3 — Scan Coverage

Every market scan cycle — what the bot was watching and the regime it saw.

```json
{
  "ts": "2026-04-18T14:35:00.000000+00:00",
  "agent_id": "your-agent-id",
  "agent_name": "YourBotName",
  "pairs_scanned": ["BTC", "ETH", "SOL", "SUI", "AVAX", "DOGE", "XRP"],
  "pairs_count": 7,
  "regime": "ranging",
  "btc_dominance": 57.4,
  "market_notes": "BTC.D elevated — alt season penalty active"
}
```

## T4 — Strategy Refinement

Synthesized analysis built from T1 + T2 + score attribution. Generated by `build_t4_pods.py` on a slower cadence (2-3x/week).

```json
{
  "agent": "YourBotName",
  "epoch_id": "20260418-1800",
  "pod_type": "tier4_strategy_refinement",
  "summary": {
    "trades_analyzed": 79,
    "near_misses_analyzed": 1690,
    "loss_patterns": 44,
    "recommendations": 4
  },
  "loss_patterns": [
    {"pattern": "EMA stack mixed (1H choppy)", "occurrences": 18},
    {"pattern": "CVD falling — sell pressure", "occurrences": 14}
  ],
  "near_miss_analysis": {
    "total": 1690,
    "avg_score": 4.57,
    "max_score": 5.9,
    "top_pairs": [["BTC", 312], ["ETH", 287], ["SOL", 201]]
  },
  "recommendations": [
    "EMA mixed at entry correlates strongly with losses — add penalty",
    "High CVD sell pressure at entry is a reliable loss predictor"
  ]
}
```

## Pod envelope (all tiers)

Every pod wraps its tier data in a standard envelope:

```json
{
  "agent": "YourBotName",
  "agent_id": "your-agent-id",
  "epoch_id": "20260418-1800",
  "built_at": "2026-04-18T18:00:08.000000+00:00",
  "hours": 48,
  "data_schema": "reppo-pod-v1.0",
  "pod_type": "tier1_executed_trades",
  "summary": { ... },
  "trades": [ ... ]
}
```
